For the technical folk out there: The website's back-end is JAVA-based and is organized using the Spring Model-View-Controller design pattern. Using REST-ful web techniques, I designed the overall communication logic between the client and server in such a way that the client-side sends HTTP GET and POST requests via AJAX to the server, which then returns a JSON response back to the client to be used for various purposes. Thus, there exists a meaningful division (commonly referred to as the separation of concerns) between front-end presentation and back-end business logic which are essentially linked by HTTP Request/Responses. The back-end communicates with a MySQL database using the JDBC Adapter and uses Hibernate for Object/Relational Mapping of JAVA Object Models.
